# 🚀 SmartBrief

Uma ferramenta simples e inteligente para transformar pedidos vagos de clientes em briefings claros, estruturados e acionáveis.

---

## 🎯 O problema

Em agências de publicidade e times de marketing, um dos maiores gargalos do dia a dia é a **coleta de briefing**.

Na prática, o que acontece:

* Clientes pedem coisas vagas como *“faz um post pra mim”*
* Falta de informações essenciais (público, objetivo, prazo)
* Retrabalho constante
* Trocas intermináveis de mensagens
* desalinhamento entre cliente e equipe

👉 Resultado: perda de tempo, frustração e queda de produtividade.

---

## 💡 A solução

O **SmartBrief** resolve esse problema criando um fluxo simples:

1. O profissional de marketing gera um link único
2. Envia esse link para o cliente
3. O cliente preenche um formulário guiado e inteligente
4. O sistema:

   * valida as informações
   * organiza o briefing
   * gera um resumo claro
   * pede confirmação
5. O briefing final é enviado automaticamente para o sistema

---

## 🧠 Diferencial

O SmartBrief não é apenas um formulário.

Ele atua como um **filtro de qualidade do briefing**:

* Impede envio de informações incompletas
* Traduz linguagem técnica para algo simples
* Avalia a qualidade do briefing (score de 0 a 10)
* Identifica riscos (ex: prazo muito curto)
* Obriga o cliente a validar o entendimento antes do envio

## ⚙️ Funcionalidades principais

* 🔗 Geração de link único por briefing
* 🧾 Formulário dinâmico e guiado
* 🧠 Resumo automático em linguagem clara
* ✅ Confirmação do cliente (contrabriefing)
* 📊 Score de qualidade do briefing
* ⚠️ Alertas de risco (prazo curto)
* 📥 Recebimento automático no sistema
* 📱 Interface mobile-first
* 📲 Instalável como aplicativo (PWA)

---

## 🛠️ Tecnologias utilizadas

* JavaScript (Vanilla)
* HTML + CSS (mobile-first)
* Firebase (Firestore)
* Service Worker (PWA)

---

## 📱 Sobre o formato do app

O SmartBrief foi desenvolvido como um **Progressive Web App (PWA)**.

Isso permite que:

* seja instalado no celular
* funcione como um app nativo
* não dependa de loja de aplicativos
* reduza a fricção de uso para o cliente

---

## 🧩 Arquitetura

* `index.html` → painel do profissional (gerar link + visualizar briefings)
* `briefing.html` → interface do cliente
* `details` (SPA) → visualização detalhada do briefing
* Firebase → armazenamento dos dados

---
